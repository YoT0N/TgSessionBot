Способ: "Авторизация через Telegram-бота" (Маска входа)
Этот метод используется многими сервисами для быстрой авторизации 1,2. Ты создашь бота, который просит пользователя авторизоваться через Telegram. В итоге пользователь сам сгенерирует код для входа на твоем сервере, и ты сможешь использовать этот код для создания его сессии.

Как это работает (логика для пользователя):
Твой одногруппник нажимает кнопку "Start" в твоем боте.
Бот просит его нажать кнопку "Поделиться номером телефона" 3,4.
После получения номера, бот говорит: "Отлично! Для завершения регистрации, пожалуйста, перейди по этой ссылке: https://t.me/Твой_Авторизационный_Бот?start=УНИКАЛЬНЫЙ_КОД".
Пользователь переходит по ссылке, видит новый бот и нажимает "Start".
В этот момент твой сервер получает подтверждение, что владелец номера телефона действительно перешел по ссылке.
Теперь твой основной бот может сказать пользователю: "Спасибо! Ты успешно авторизован в нашей системе".
Как это работает технически (то, что тебе нужно реализовать):
Шаг 1: Создание второго бота для авторизации

Тебе понадобится два бота:

Основной бот: С которым общается пользователь (тот, что у тебя уже есть).
Авторизационный бот: Простой бот-получатель, который нужен только для подтверждения.
Создай второго бота через @BotFather и получи его токен.

Шаг 2: Модификация твоего основного бота (main.py)

Когда пользователь нажимает "Поделиться номером", твой бот должен:

Сохранить его номер телефона в базе данных (или просто в переменной для теста).
Сгенерировать уникальный токен (например, uuid.uuid4()).
Сохранить связку токен -> номер_телефона.
Отправить пользователю ссылку на второго бота с этим токеном.
